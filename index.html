Served allows users to input current ingredients to generate recipes based on what they already have at home. Each recipe generated from their selected ingredients shows a thumbnail of the recipe, the percentage of which it matches the ingredients the user has listed and a heart button at the bottom of the card to "save" their favorite recipes for later use.
Saved recipes are reachable with the button in the upper right section of the page. There are also buttons in the navigation menu to see the developers and report bugs/issues with the site.

This app is a simulated bookstore run on a Java console window, making prolific use of the java.util.Scanner class. Users can register new accounts, login, make offers on books, view owned items, and make payments. Employees can add/remove items, edit item details, and view payments.

This app is a wedding budget planner, allowing the user to create budget items, budget categories, to-do lists, and see their data displayed as estimated/actual costs through pie, bar and line graphs. Users can edit, delete, and add these items, which route to an API built with Express, Node, and Sequelize.
The MySQL database holding user data is run on my computer through MySQL Workbench, and deployed to Heroku with JawsDB. The graphs are built using Chart.js, accounts are created with user authentication via Passport.js, and repository updates are screened by ESlint and Travis CI before being pushed to the main branch.
